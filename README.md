# Raspberry Pi Monitor

Lightweight web-based system monitor for Raspberry Pi. Single-command install, browser dashboard, REST JSON API.

## Features

- CPU temperature with throttling status
- CPU frequency
- Load average (1/5/15 min)
- RAM and disk usage
- Network traffic per interface
- Active TCP connection count
- Available system updates
- Top 15 processes by CPU
- Token-based auth (Bearer)
- Optional `/reboot` and `/shutdown` endpoints (passwordless sudo, opt-in)
- Cached snapshot (5s TTL by default)

## Requirements

- Raspberry Pi OS (Bookworm or later recommended)
- Node.js 20+
- PM2

## Installation

```bash
curl -fsSL https://raw.githubusercontent.com/erresoftware/rpi-monitor/main/install.sh | bash
```

The installer prints a randomly generated `AUTH_TOKEN` at the end. Save it.

## Usage

Open `http://<rpi-ip>:3002` and paste the token when prompted.

### REST API

```
GET /api               — JSON snapshot (Bearer auth)
POST /reboot           — reboot device (Bearer auth, opt-in)
POST /shutdown         — shutdown device (Bearer auth, opt-in)
```

Auth header:

```
Authorization: Bearer <AUTH_TOKEN>
```

## Configuration

Edit `~/rpi-monitor/.env`:

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | `3002` | HTTP port |
| `HOST` | `0.0.0.0` | Bind address |
| `AUTH_TOKEN` | _required_ | Bearer token (generated by installer) |
| `ENABLE_POWER_ENDPOINTS` | `false` | Enable `/reboot` and `/shutdown` |
| `CACHE_TTL_SECONDS` | `5` | Snapshot cache TTL |

After editing, restart:

```bash
pm2 restart rpi-monitor
```

## Enabling power endpoints

1. Set `ENABLE_POWER_ENDPOINTS=true` in `.env`.
2. Add a sudoers rule (replace `pi` with your user):

   ```
   pi ALL=(root) NOPASSWD: /sbin/reboot, /sbin/shutdown
   ```

   Save to `/etc/sudoers.d/rpi-monitor` via `sudo visudo -f /etc/sudoers.d/rpi-monitor`.
3. Restart the service.

The service never receives or forwards a sudo password — it relies on the NOPASSWD rule scoped to those two binaries only.

## Security notes

- Bind to `127.0.0.1` instead of `0.0.0.0` if you only need local access, then expose via a reverse proxy with TLS.
- Behind a reverse proxy (Caddy, Nginx), prefer HTTPS with a self-signed or Let's Encrypt cert.
- The included rate limit is conservative; tune `apiLimiter` and `powerLimiter` in `src/index.js` for your environment.
- Power endpoints are disabled by default. Enable only after the sudoers rule is in place.

## Uninstall

```bash
pm2 delete rpi-monitor
pm2 save
rm -rf ~/rpi-monitor
sudo rm -f /etc/sudoers.d/rpi-monitor
```
